notebooks/05_esg_climate.ipynb  — STEP 7 ADDED
  Two new cells inserted after the existing ESG triangulation section:

  Markdown cell (Step 7 header):
    Explains that Steps 1–6 produce Bloomberg-based preliminary scores and
    Step 7 overwrites them with the specialist's FactSet triangulation.

  Code cell (Step 7 implementation):
    Reads Portfolio_Screening_Output.xlsx (all sheets), loads the latest
    master_dataset_*.csv for ticker mapping, normalises company names with
    regex suffix-stripping, joins Stage 1 and Stage 2 data to master tickers,
    min-max normalises E/S/G pillar z-scores to 0–100, sets ESG_score to
    in_house_pct (0–100 percentile) for Stage 1 PASS companies and 0 for
    excluded/hard-excluded companies, imputes missing carbon intensity from
    sector median, and saves outputs/scores/esg_scores_YYYY-MM-DD.csv.

    NB10 auto-picks the latest esg_scores_*.csv file (glob + sort), so
    running Step 7 is sufficient — no path changes needed in NB10.

  WHY the change: The original triangulation used (a) Bloomberg ESG
  Disclosure score — same methodology family as in-house score, not
  independent — and (b) yfinance Sustainalytics, which Yahoo Finance removed
  (always returned 0). The FactSet approach uses three genuinely independent
  sources: Truvalue SASB AI, Sustainalytics, and ISS.
