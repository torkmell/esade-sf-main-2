----------------------------------------------------------------------
PREREQUISITES (ONE-TIME SETUP)
----------------------------------------------------------------------

1. Run notebook 10 first so final_portfolio_*.csv exists in outputs/portfolio/
2. From the project root, run the preparation script:

       python scripts/prepare_rag_inputs.py

   This creates the 20 company folders and prompt files in data/rag/reports/
   and initialises the tracking files in outputs/rag/.

----------------------------------------------------------------------
HOW TO RUN THE 8-TEST FOR ONE COMPANY  (repeat for all 20)
----------------------------------------------------------------------

STEP 1 — OPEN THE COMPANY PROMPT FILE
  Go to:  data/rag/reports/TICKER_Company_Name/TICKER_8test_prompt.txt
  Example: data/rag/reports/19V_VAT_Group_AG/19V_8test_prompt.txt

  The file is divided into 7 steps. Read through it before starting.

STEP 2 — COLLECT AND UPLOAD TIER 1 DOCUMENTS
  Tier 1 = primary company documents (sustainability report, annual report,
  TCFD, CSRD). The prompt file lists what to find under STEP 1.

  - Download the PDFs from the company's investor relations page.
  - MANDATORY: name files starting with tier1_ before saving:
      tier1_sustainability_report_2024.pdf
      tier1_annual_report_2024.pdf
      tier1_climate_report_2024.pdf
  - Save to the company folder:
      data/rag/reports/TICKER_Company_Name/
  - Upload these PDFs to a dedicated Claude Project for this company.
    (One Claude Project per company, or one project for multiple companies
    if you run out of project slots.)

STEP 3 — COLLECT AND UPLOAD TIER 2 SOURCES
  Tier 2 = external verification (SBTi, CDP, news investigations).
  Use to challenge or corroborate Tier 1 claims — not as primary evidence.

  - MANDATORY: name files starting with tier2_ before saving:
      tier2_sbti_dashboard_2024.pdf
      tier2_cdp_response_2024.pdf
      tier2_ngo_report_2024.pdf
  - Save to the same company folder and upload to the same Claude Project.

STEP 4 — PASTE THE ANALYSIS PROMPT INTO CLAUDE PROJECTS
  Inside TICKER_8test_prompt.txt, find the section marked:

       STEP 5 — PASTE THIS PROMPT INTO CLAUDE PROJECTS

  Copy everything from that section header down to the closing JSON schema.
  Paste it into the Claude Projects chat for that company and send it.

  Claude will return a JSON block with ratings for all 8 dimensions.

STEP 5 — SAVE THE JSON OUTPUT
  Copy the JSON output from Claude Projects.
  Save it to:  outputs/rag/greenwash_TICKER.json
  Example:     outputs/rag/greenwash_19V.json

  (Notebook 09 reads this file automatically.)

STEP 6 — ENTER RATINGS IN THE RAG WORKBOOK  ← MOST IMPORTANT STEP
  Open:   data/rag/RAG_Screening_Sheet_Workbook_v1.xlsx
  Sheet:  RAG Screening Sheet
  Find the row for your company (match Bloomberg ticker in column B).

  Fill in the following columns using PASS / PARTIAL / FAIL / MISSING:
    CJ  8-Test 1: Specificity
    CK  8-Test 2: Metric
    CL  8-Test 3: Baseline
    CM  8-Test 4: Target
    CN  8-Test 5: Time Horizon
    CO  8-Test 6: Scope
    CP  8-Test 7: Verification
    CQ  8-Test 8: Consistency

  Also fill:
    DC  Key evidence quote (verbatim quote from source + page number)
    DE  Evidence page reference
    DX  Verdict rationale (2-3 sentences)
    DF  Evidence Confidence (1 = low, 5 = high)
    DH  Verification Status (VERIFIED / UNVERIFIED / PENDING)

  Save the workbook after each company. The workbook is the authoritative
  human-reviewed layer — automation never overwrites your entries.

STEP 7 — VERIFY YOUR WORK (QUALITY CONTROL)
  The project requires:
    - 30% of all extractions manually verified against the source PDF
      (check quote + page number match the actual document)
    - 100% of any company on the watchlist or exclusion list verified

  To check which companies need verification:
  Open outputs/rag/rag_screening_summary.csv — column verification_pct
  shows how much of each company has been checked.

----------------------------------------------------------------------
RUNNING NOTEBOOK 09  (after all 20 companies are filled in)
----------------------------------------------------------------------

Once the RAG workbook contains ratings for all 20 portfolio companies,
run notebook 09 (notebooks/09_greenwashing.ipynb) in order.

The notebook:
  1. Reads the workbook ratings
  2. Translates PASS/PARTIAL/FAIL/MISSING → LOW/MED/HIGH/MISSING
  3. Counts HIGH flags per company
  4. Applies exclusion rule:  ≥ 3 HIGH flags → excluded
  5. Applies watchlist rule:  exactly 2 HIGH flags → watchlist
  6. Saves outputs/scores/greenwashing_scores_DATE.csv
  7. Appends one row per company to outputs/rag/portfolio_change_log.csv

If a company is excluded, notebook 10 (portfolio construction) will
replace it with the next-best candidate from the ranked universe.

----------------------------------------------------------------------
PORTFOLIO CHANGES: WHAT HAPPENS TO THE FOLDERS
----------------------------------------------------------------------

The 20 company folders in data/rag/reports/ are created by the
preparation script based on whoever is in the current portfolio CSV.

If the portfolio changes (a company is swapped out):
  - Re-running the script adds folders for NEW companies.
  - Folders for REMOVED companies are NOT deleted automatically.
  - To keep the folder tidy: manually delete the folder for any company
    that is no longer in the final portfolio.

----------------------------------------------------------------------
TROUBLESHOOTING
----------------------------------------------------------------------

Problem: Company missing from workbook
  → Check that the company's Bloomberg ticker appears in column B of
    the RAG Screening Sheet. If missing, add the row manually.

Problem: Notebook 09 fails with KeyError
  → The column names in the workbook do not match what the code expects.
    Open the workbook and confirm row 6 contains the column headers
    (not row 4 or 5). Contact the Pipeline Operator.

Problem: JSON output from Claude Projects is malformed
  → Paste it into https://jsonlint.com to identify the error.
    Common fix: remove trailing commas or fix mismatched brackets.

Problem: Company name or ticker is wrong in the prompt file
  → Re-run:  python scripts/prepare_rag_inputs.py
    The script regenerates all prompt files from the latest portfolio CSV.

----------------------------------------------------------------------
KEY RULES  (do not break these)
----------------------------------------------------------------------

1. NEVER edit the workbook formula cells (columns A, B pulled from
   Universe Tracker). Only edit the data entry columns listed above.

2. NEVER paste AI-generated ratings directly without reading the source.
   Claude Projects can hallucinate. Always spot-check quotes vs PDF.

3. NEVER mark a company MISSING on all 8 dimensions without noting why
   in the Verdict Rationale column — the examiner will ask about it.

4. The workbook is the single source of truth for greenwashing ratings.
   JSON files and CSV outputs are derived from it, not the other way.
